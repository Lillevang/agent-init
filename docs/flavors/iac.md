# iac

A flavor for Infrastructure-as-Code repos. One combined scaffold covers Terraform, Ansible, or both — recipes auto-detect which toolchain is in use and no-op gracefully when a tool is missing or absent from the repo. Source: [internal/flavors/iac/](../../internal/flavors/iac/).

## Heads-up: scope difference from the roadmap

The project roadmap in [`CLAUDE.md`](../../CLAUDE.md) originally listed `terraform` and `ansible` as two separate planned flavors. They are intentionally shipped as one `iac` flavor instead. The reason: real-world platform repos almost always have both — Terraform provisions VPCs/clusters/IAM and Ansible configures the resulting hosts. Maintaining two near-identical devcontainers and two copies of the same `.yamllint.yml`/`.pre-commit-config.yaml`/`Justfile.tmpl` would have meant more drift than reuse.

The `Justfile` and codemap script handle the "Terraform-only" and "Ansible-only" cases explicitly, so if you really do have a pure-Terraform or pure-Ansible repo the flavor still works without dead recipes complaining.

## Usage

```bash
agent-init init iac path/to/new-repo
```

After scaffolding:

1. Edit `terraform/versions.tf` to pin the providers you actually use and configure your remote backend.
2. Edit `.devcontainer/devcontainer.json` and **uncomment the credential mounts** you need (`~/.aws`, `~/.config/gcloud`, `~/.azure`, `~/.ssh`). They are commented by default to make the security tradeoff explicit.
3. Edit `.agent/AGENTS.md` to describe THIS project's environments, state backend, and change-management process.
4. `devcontainer up` and run `just check` to confirm the done-gate passes.

## Justfile recipes

The recipes use `command -v` guards plus directory/file existence checks so the same Justfile works in a Terraform-only repo, an Ansible-only repo, and a mixed one. None of them fail when a tool is missing — they print a `skipping` notice and exit 0.

| Recipe | What it does |
|--------|--------------|
| `fmt` | `terraform fmt -recursive` if any `*.tf` exists |
| `lint` | `tflint --recursive` (if `.tflint.hcl` + `*.tf`), `ansible-lint` (if `roles/` or `playbooks/`), `yamllint .` |
| `typecheck` | `terraform init -backend=false && terraform validate` per module; `ansible-playbook --syntax-check` per playbook |
| `test` | `terraform test` if `tests/` exists; `molecule test` if `molecule/` exists |
| `security` | `tfsec .` and `trivy config .` — advisory, **not** invoked by `check.sh` |
| `tf-init` | `terraform init -backend=false` in every directory containing `*.tf` (useful after pulling new modules) |
| `ansible-deps` | `ansible-galaxy install -r requirements.yml` (or `ansible/requirements.yml`) |

Recipe definitions: [internal/flavors/iac/templates/Justfile.tmpl](../../internal/flavors/iac/templates/Justfile.tmpl).

## Codemap

[gen-codemap.sh](../../internal/flavors/iac/templates/.agent/scripts/gen-codemap.sh) overrides the common script. It surfaces:

- Directory tree (with `.terraform/` and `.venv/` excluded).
- Terraform modules under `modules/` or `terraform/modules/` (with `.tf` file counts).
- Terraform `variable` / `output` / `resource` / `module` / `data` declarations.
- Ansible roles under `roles/` or `ansible/roles/` (with task counts from `tasks/main.yml`).
- Ansible playbooks under `playbooks/` or `ansible/playbooks/` (with task counts).
- Stats: tracked file count, `.tf` count, Ansible `.yml` count.

This is what an IaC agent actually wants to navigate by — Go API surface is not the right shape.

## Security model

The whole flavor is built around three concerns that go-cli / go-backend don't have:

1. **State files are credentials.** `.gitignore` blocks `*.tfstate*` and `.terraform/`. AGENTS.md explicitly requires a remote backend before any shared-environment `apply`.
2. **Cloud credentials live on the host.** The devcontainer mounts `~/.aws`, `~/.config/gcloud`, `~/.azure`, and `~/.ssh` **read-only** — and all four mounts are **commented out by default**. The user opts into each one consciously.
3. **Real inventory leaks topology.** `inventory/hosts.yml` is gitignored; only `inventory/hosts.yml.example` is committed.

The AGENTS.md "Hard rules" section ([file:25-69](../../internal/flavors/iac/templates/.agent/AGENTS.md.tmpl#L25-L69)) spells these out in agent-readable form.

## Template files and the Jinja gotcha

Ansible YAML uses Jinja2 `{{ var }}` natively, which collides with Go's `text/template` syntax. The fix is to keep playbook and inventory files as plain `.yml` (not `.tmpl`) so the scaffold engine copies them verbatim. The only files in this flavor with `.tmpl` extensions and content templating are:

- `.agent/AGENTS.md.tmpl` — uses `{{.ProjectName}}`. The one literal `{{ var }}` reference in the prose is escaped as `{{"{{"}}`.
- `Justfile.tmpl` — uses `{{.ProjectName}}` in the header comment.
- `.devcontainer/devcontainer.json.tmpl` — uses `{{.ProjectName}}` for the container and workspace names.
- `terraform/main.tf.tmpl` — uses `{{.ProjectName}}` for the example `locals.project`.
- `README.agent.md.tmpl` — uses `{{.ProjectName}}` throughout.

`ansible/playbooks/site.yml` and `ansible/inventory/hosts.yml.example` are deliberately **not** templates — they would otherwise need every Jinja `{{ }}` escaped, which would be miserable to maintain.

## Tests

[test/golden_test.go](../../test/golden_test.go) covers `iac` in the table-driven `TestFlavorGolden`. The golden snapshot at [testdata/golden/iac/](../../testdata/golden/iac/) is byte-compared against a fresh scaffold; regenerate via `just smoke-test-update` after intentional template changes.

The smoke-test recipe also runs `just check` inside the scaffolded output. Because the recipes are guarded with `command -v`, the gate passes on a CI runner that doesn't have terraform/ansible/tflint installed — only the codemap step actually executes work. The full toolchain runs in the devcontainer.

## Edge cases

- **No tools installed.** Every recipe skips with a notice and exits 0. The done-gate passes but does nothing useful — that's the right default for a fresh scaffold on a bare host.
- **Pure-Terraform repo.** Drop `ansible/`, `.ansible-lint`, and `ansible.cfg`. The Justfile's Ansible branches are skipped because the directories don't exist.
- **Pure-Ansible repo.** Drop `terraform/` and `.tflint.hcl`. Same logic in reverse.
- **`pre-commit-terraform` and `ansible-lint` pre-commit hooks** are heavy. If commits drag, move `ansible-lint` to `stages: [pre-push]`.
- **Provider plugin downloads** in `terraform init` are slow and bandwidth-heavy. The `post-create.sh` runs `terraform init -backend=false` once per `*.tf` directory at container build time so subsequent `validate` calls are instant.
